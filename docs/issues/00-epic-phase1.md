# Epic: Phase 1 — 先勁藥局 pilot(LINE 服藥提醒 + 續領留客)

## Context

先勁藥局約 50 位病人。慢性病/長輩服藥依從性低、斷藥不回來續領,藥局缺低成本追蹤工具。Phase 1 交付一條完整迴圈:藥師建處方 → 病人 LINE 收提醒 → 一鍵回報 → 藥局看到「快斷藥沒回來」的留客儀表板並一鍵召回。賣點是**續領率**,不是按鈕自陳依從。

⚠️ **開工前提:** Phase 0(留客儀表板 mockup + 老闆會議)需先拿到「續領率願付數字 + 病人名單」。本 epic 是備好的工料。

## Dependency graph

```
01 Foundation ─┬─> 02 Model+Decomposer ─┬─> 03 Scheduler ─┬─> 05 Refill loop (E1)
               │                         ├─> 04 Confirm+Stats ─┐
               │                         └─> 07 LIFF form       └─> 08 Dashboard
               └─> 06 Image+Consent (independent)
```

## Sequencing rationale

01 是地基(沒 webbook 收不到任何事件)。02 的 dose_event 模型是 03/04/05/07/08 全部的前提。03 排程器 + 04 確認可平行(都只依賴 02)。05 續領鉤依賴 02(處方天數)+ 03(排程)。06 影像獨立,可任何時候平行。08 儀表板最後接 04+05 的真資料。

## Definition of Done

1. 一位測試病人走完:加好友 → 藥師建一張 TID 7 天處方 → 收到每餐提醒 → 按已服藥計數 +1、重複按不重複 → 逾時觸發升級 → 天數將盡收到續領提醒 → 藥局儀表板看到此人狀態。
2. A1 簽章驗證、A3 推播重試、A8 冪等防重複 全部有測試。
3. 排程器重啟後未到期提醒仍會發(持久化佇列證明)。
4. 處方影像存私有桶,無公開可讀 URL;病人有同意紀錄。

## Out of scope(Phase 1 不做)

- 家屬共享儀表板、原生 App 鬧鐘/鳴笛、多藥局後台、保險/健保串接。
- 回診提醒(綁診所資料)— 留 Phase 2。
